What is Eleventy?

  • it's a static site generator
  • creates a website from your content :D

The minimal example

echo "# Hello World!" > index.md
npx @11ty/eleventy

but let's do something more exciting :)

Create a new project

mkdir my-awesome-site
cd my-awesome-site
git config --global init.defaultBranch main
git init
npm init -y
npm i @11ty/eleventy -D

.gitignore

node_modules
.DS_Store
Thumbs.db

# Eleventy output folder
public

.eleventy.js Config

module.exports = (config) => {
  // specify folders to be copied to the output folder
  config.addPassthroughCopy('./src/js/');
  config.addPassthroughCopy('./src/css/');

  return {
    dir: {
      input: 'src',    // default: '.'
      output: 'public' // default: '_site'
    }
  }
};

package.json

{
  "scripts": {
    "start": "eleventy --serve",
    "build": "eleventy"
  }
}

Content: src/index.md

---
title: "Hello world"
---
# Hello World

Welcome to my awesome {{title}} site!

Run and Build

  • npm start -> Development server
  • npm run build -> build it

From markdown to paths

For each markdown content file, eleventy creates a folder with an index.html for nice urls:

index.md            --> /
about.md            --> /about/
faq.md              --> /faq/
blog/hello-world.md --> /blog/hello-world/

Layout templates

src/_includes/base.njk

<!DOCTYPE html>
<html lang="en">
  <head><title>{{ title }}</title></head>
  <body>
    <main>
      {% block content %}
        {{ content | safe }}
      {% endblock %}
    </main>
  </body>
</html>

Extending layouts

{% extends "base.njk" %}

{% block content %}
  <article class="article">
    {{ content | safe }}
  </article>
{% endblock %}

Includes

You can include partial layouts anywhere(*) in your njk or markdown files:

{% include 'header.njk' %}

*to make includes work from inside markdown files, add templateEngineOverride: njk,md to the front matter

Everything about Nunjucks

  • https://mozilla.github.io/nunjucks/

Other supported formats:

.html, .liquid, .hbs, .ejs, .haml, .pug

Providing data for your site

  • file specific: in the markdown's front matter
  • folder specific: add a json file to a content folder
  • globally _data directory: globally available
  • _data supports .js, .yaml, .json files

_data example

src/_data/nav.json

[
  {"title": "Home", "url": "/"},
  {"title": "Blog", "url": "/blog/"}
]

_data example part 2

src/_includes/nav.njk

<nav>
  <ul>
    {% for link in nav %}
      <li>
        <a href="{{ link.url }}">{{ link.title }}</a>
      </li>
    {% endfor %}
  </ul>
</nav>

_data javascript example

_data/site.js

module.exports = {
  name: 'My awesome site',
  url: 'https://awesome.site/'
};

Can be used like this in the content: {{ site.name }}

Collections

You can tag your content with a keyword and then iterate through these via collections.

This is useful for auto-generating table of contents or listing articles that are related to each other

Collections example

In your src folder, add a blog folder with a bunch of markdown files. Tag them as posts:

---
tags: posts
---

Collections example part 2

index.md

# Blog

<ul>{% for post in collections.posts %}<li>
  <a href="{{ post.url }}">
    {{ post.date | date: '%Y-%m-%d' }}: {{ post.data.title }}
  </a>
</li>{% endfor %}</ul>
